# Issue #184: empty project_id should return 400

## Summary

The project API should return a deterministic bad-request response when
`project_id` is empty.

## Reproduction

```bash
go test ./...
```

The regression test `TestGetProjectRejectsEmptyProjectID` checks that
`GetProject("")` returns a validation error before the store lookup.

## Expected Behavior

`GetProject("")` should return:

- `StatusCode: 400`
- a non-nil validation error
- no store lookup requirement

## Implemented Fix

`internal/api/projects.go` contains a small early guard:

```go
if projectID == "" {
	return Response{StatusCode: StatusBadRequest, Err: errors.New("project_id is required")}
}
```

The rest of the handler behavior should remain unchanged.
